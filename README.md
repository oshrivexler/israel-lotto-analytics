# לוטומטריקס — מנוע ניתוח וחיזוי ללוטו הישראלי (6/37)

דשבורד full-stack לניתוח סטטיסטי וחיזוי של הגרלות הלוטו של מפעל הפיס.
ממשק עברית מלא (RTL) באסתטיקת "trading desk" כהה.

## ארכיטקטורה

```
analysis/analyze.py    ← מנוע הנתונים הראשי (Python / Pandas) — קורא CSV, מחשב הכל, כותב stats.json
analysis/analyze.mjs   ← פורט שקול ב-Node.js (גיבוי) — פלט זהה בייט-אחר-בייט
src/data/stats.json    ← הפלט המחושב (נוצר ע"י analyze)
src/                   ← דשבורד React + Tailwind CSS v4 (RTL)
```

> הקליטה נעשית עם **Pandas** (`pd.read_csv` בקידוד windows-1255). שתי הגרסאות (Python ו-Node)
> חולקות אותו מחולל פסאודו-אקראי דטרמיניסטי (mulberry32) ולכן מייצרות בדיוק את אותם 5 צירופים.
> אומת באמצעות deep-diff: **0 הבדלים** בין הפלטים.

## שכבת הבהירות והנגישות (UX)

האתר תוכנן להיות מובן גם למי שאינו מבין גרפים/סטטיסטיקה:
- **`IntroBanner`** — משפט-על כן בראש הדף ("מה האתר עושה" + "האמת על לוטו") ופס הבהרה עליון.
- **`Explainer`** — תחת כל גרף: "מה רואים / המסקנה / ההמלצה" בשפה פשוטה (המלצות כנות, ללא הבטחת זכייה).
- **`Term`** — Tooltips נגישים על מונחים (מרקוב, תוחלת, נקודת מתיקה).
- **`Glossary`** — מודאל מילון מונחים מרוכז (כפתור צף, נסגר ב-Esc).
- **חשיפה הדרגתית** — ניתוחים מתקדמים מקופלים מאחורי כפתור, להפחתת עומס.
- **נגישות** — תמיכת RTL, ניווט מקלדת, קישור "דלג לתוכן", צבעים מכויילים לניגודיות WCAG, ו-`prefers-reduced-motion`.

## כלים אינטראקטיביים

- **`DrawLookup`** — חיפוש תוצאת הגרלה לפי מספר (כל ההגרלות בשני העידנים).
- **`CheckNumbers`** — "בדוק את המספרים שלך": מנתח צירוף שהוזן (סכום, זוגי/אי-זוגי, עוקבים),
  נותן **מדד פופולריות** (סיכוי לחלוק פרס) ובודק אם הצירוף המדויק יצא אי-פעם בהיסטוריה.
- **`Simulator22`** — סימולטור 22 שנה: כמה היית פוגע ב-3/4/5/6 אם היית משחק אותו צירוף בכל הגרלה,
  כמה היית מוציא, ומסקנה מפכחת על הסיכויים האמיתיים.
- **`NextDraw`** — טיימר חי להגרלה הבאה (מחושב לפי לוח ההגרלות שלישי/שבת), סכום הפרס וקישור לאתר פיס.
  הגדרות ב-`src/data/next-draw.json` (סכום הפרס לעדכון ידני — נתוני פיס החיים נטענים אצלם ב-JS ולא ניתנים למשיכה אוטומטית).
- **`ModelsLegend`** — מקרא מתקפל שמסביר בפשטות את 4 המדדים ו-5 השיטות שמאחורי התחזיות.

> הכלים נשענים על `src/data/draws.json` — מאגר חיפוש של כל ההגרלות (נוצר ע"י מנוע הניתוח).
>
> **שפה נגישה:** מונחי הז'רגון הוחלפו בשפה יומיומית ("נקודת מתיקה" → "הטווח הנפוץ", "תוחלת" → "הממוצע הצפוי",
> "פיגור" → "מספרים שלא יצאו מזמן", "שרשרת מרקוב" → "לפי ההגרלה הקודמת").

## מה המנוע מחשב

1. **קליטה ופענוח** — קריאת CSV בקידוד `windows-1255`, מיפוי 6 מספרים (1–37) + מספר חזק (1–7),
   טיפול באנומליות, מיון מהישן לחדש.
2. **סינון עידן** — הקובץ מכיל פורמטים היסטוריים שונים (לוטו ישן 6/49). המנוע מזהה אוטומטית
   את תאריך המעבר לפורמט המודרני (09/03/2004) ומנתח רק את 2,477 ההגרלות בפורמט 6/37.
3. **תדירות** — חמים/קרים בכל הזמנים, 5 שנים, שנה אחרונה.
4. **פיגור (Overdue)** — מספר ההגרלות מאז ההופעה האחרונה של כל מספר.
5. **נקודות מתיקה** — התפלגות סכום (ממוצע±סטיית-תקן), יחס זוגי/אי-זוגי, זוגות עוקבים.
6. **שרשרת מרקוב** — הסתברויות מעבר מותנות בהינתן ההגרלה האחרונה.
7. **תיק תחזיות** — 5 צירופים מאוזנים (חם/קר/מרקוב) הנופלים בתוך נקודות המתיקה.

## הרצה

```bash

pip install pandas numpy
npm install

npm run analyze       # מריץ את מנוע ה-Python ומדפיס דו"ח אימות
npm run analyze:node  # חלופה: מנוע ה-Node (פלט זהה)
npm run dev           # שרת פיתוח (http://localhost:5180)
npm run build         # analyze (Python) + בנייה לפרודקשן
```
